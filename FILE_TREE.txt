QUICK START GUIDE

1. LOCAL DEVELOPMENT:
   ├── ./setup.sh              (One-time setup)
   ├── redis-server &          (Terminal 1)
   ├── ./run.sh                (Terminal 2)
   └── http://localhost

2. DOCKER DEPLOYMENT:
   ├── docker compose up --build
   └── http://localhost

═══════════════════════════════════════════════════════════════

KEY FILES EXPLAINED

Backend Flow:
  main.py → routers → openrouter_client.py → OpenRouter API
                   → redis_client.py → Redis

Frontend Flow:
  SmartLens_HK_Final.html → JavaScript → /api/* → Backend
                                      → /api/notifications/stream (SSE)

Config Flow:
  .env.example → .env (user edits) → config.py → Throughout app

═══════════════════════════════════════════════════════════════

DEPLOYMENT OPTIONS

Option 1: Docker (Production)
  └── docker-compose.yml (Redis + Backend + Nginx in one command)

Option 2: Local (Development)
  ├── Redis (separate)
  ├── Backend (./run.sh or uvicorn)
  ├── Nginx (separate or skip)
  └── Frontend (served by Nginx or locally)

API ENDPOINTS

POST /api/calories/analyse       Upload food image → nutrition
POST /api/news/analyse           Classify headline → fake/real
POST /api/health/steps           Daily steps → insights
POST /api/health/sleep           Sleep hours → analysis
GET  /api/notifications/stream   Real-time notifications (SSE)
GET  /health                     Health check

═══════════════════════════════════════════════════════════════

CACHE STRATEGY

Cache Type          Duration  Purpose
─────────────────────────────────────────
Food images         6 hours   Nutritional profiles don't change
News analysis       1 hour    Stories evolve slower
Step insights       10 min    Daily activity changes quickly
Sleep analysis      1 hour    Sleep pattern changes daily
Notifications       None      Real-time (SSE stream)
